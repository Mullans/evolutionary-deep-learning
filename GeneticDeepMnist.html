Current Progress:¶

  • Models are generated with random parameters for a pre-set number of layers. Parameters include number of nodes per layer, activation functions for each layer, dropout (applied after activation functions) for each layer, and the batch size for each model.
  • At the end of each generation, pairs of models are randomly selected and "bred" to create a new child model. Each parameter of the child model will come from one of the parent models. After the child is generated based on the traits of the parents, then there is a slight chance for each of the parameters that they will randomly mutate their value.
  • Elitism is implemented, but currently only tested with 1. This keeps the best model from the previous generation as part of the next generation and replaces all of the other models with new child models.
  • The current best accuracy found after 5 generations of evolving a population of 10 models that are evaluated based on their test accuracy after 10 generations on classifying MNIST data: 97.67%.
  • Each generation can take anywhere from ~3 minutes to ~15 minutes to run at with the current hyper-parameters. THis is highly variable, because it relies on the time it takes to train each model in the population for the desired number of epochs before evaluation.

Next Steps:¶

  • Experiment with evolutionary hyper-parameters. It takes a while to train larger populations, so it may take a while to get decent results.
  • Change probabalistic cross-over to top-x crossover. This will give the top-x models the same chance at reproduction rather than weighting heavily towards only the best. This might help avoid local maxima that cause large short-term gains, but lose features that can help approach a more global maximum.
  • Implement a graph based evolutionary algorithm. By evolving based on a tree-like structure, a single population can have models of different depths and sizes. Plus, this could also allow for things like skip-connections and inception models to occur.
